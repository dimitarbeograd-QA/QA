# Тест кейсове — QA (Famous Italian Explorers app)

Легенда: **P** = Priority (P1 критичен, P2 висок, P3 среден).
Тези кейсове допълват автоматизирания suite в `tests/` — фокус върху ръчна
проверка и edge cases извън покритите автоматизирани тестове.

## GET /hello

| ID | Стъпки | Очакван резултат | P |
|---|---|---|---|
| TC-01 | Извикай `GET /hello` без query параметър | Отговор `200` с текст `hello Guest` | P1 |
| TC-02 | Извикай `GET /hello?name=xy_z` | Отговор `200` с текст `hello xy_z` | P1 |
| TC-03 | Извикай `GET /hello?name=` (празна стойност) | Определи и документирай очакваното поведение (fallback към Guest или празно име) | P2 |
| TC-04 | Извикай с name съдържащ специални символи/HTML (`<script>`) | Стойността се връща escaped, не се изпълнява като код (XSS проверка) | P1 |

## PUT /travellers

| ID | Стъпки | Очакван резултат | P |
|---|---|---|---|
| TC-05 | Изпрати `{surname: "Colombo"}` | Отговор `200`, JSON с `name: "Cristoforo"`, `surname: "Colombo"` | P1 |
| TC-06 | Изпрати `{surname: "da Verrazzano"}` | Отговор `200`, JSON с `name: "Giovanni"` | P1 |
| TC-07 | Изпрати несъществуваща фамилия | Определи очаквано поведение (404 / празен обект / грешка) и валидирай | P2 |
| TC-08 | Изпрати заявка без body / с невалиден JSON | Сървърът връща подходящ статус код на грешка, не крашва | P1 |
